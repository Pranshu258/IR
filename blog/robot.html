Space Structure Learning with ISOMAP

Learning the structure of space for a mobile robot, based on observations (i.e. images from the robot's camera) and nothing else with the help of the ISOMAP algorithm.

For this experiment, a point robot in a square shaped arena with walls having a color gradient has been used. The field of view of the robot's camera is 120 degrees. The robot can move anywhere in 'x' and 'y' directions within the arena and rotate about its own axis. The size of the robot itself has been accounted for (by keeping sufficient distance from the walls of the arena).






ROTATING ABOUT FIXED POINT(S)

We make the robot rotate about its own axis at three different points in the arena and collect images of the arena as seen by the robot.

With configuration given above, the images we get are 600 x 1 pixels in size in the RGB color sapce. So, the data we get has entries of '600 x 1 x 3'' dimensions. We capture 1000 images for each of the three points, i.e. 3000 in total. We have observed similar results on a bigger image set also.
We then find out the hidden manifold in this data with the help of the Isomap Algorithm. The algorithm takes two parameters, the number of neighbors to consider in the geodesic mapping of the manifold and the final output dimension that we want. We have provided them as '10' and '3' respectively.

DISCOVERING X-Y MOTION WITH ROTATION


In this part we make the robot move around a randomized trajectory of large length and collect about 10000 images. Then we learn the embedded manifold with the help of the Isomap Algorithm.

We have conducted this experiment on 3 trajectories:

  • Completely Random Points [x, y, Theta]
  • Motion along the Diagonal with continuous rotation about the robot's own axis
  • Circular with view in Radial direction

ESTIMATING THE POSITION OF THE ROBOT FROM TEST IMAGES

In this part, we take a set of 11000 images in which for 10000, we know exactly from what position they were captured (train set). However, we don't have this information for the rest of the images (test set).

We use the isomap algorithm to learn the embedded manifold in the dataset. Now, for each image in the test set we have its position in the 3D manifold. But, for the train set images we also have the positions from where they were captured. So, we now find out means of the nearest neighbors of test images in the train set and assign the corresponding capturing positions to the test images.

We now have all our images labelled with their respective capturing positions (x, y, theta).
**this was done for random position and the circular trajectories
**localization errors are the average euclidean displacement from the actual positions to the estimated positions
